Web设计(Head First)--笔记

发表于 2016-05-24   |   分类于 Web前端   |     |   阅读次数   |  

序言

元认知：想一想如何思考

注释：元认知（Metacognition），教育心理学专有名词。管理学习和认知的过程，增加学习效果。

  1. 更深入思考问题，注重理解记忆。
  2. 更集中注意力，从多角度考证问题最佳解。
  3. 潜意识激发对新知识的渴望，获取更多学习主动性。

驯服大脑的方法

  • 慢慢来，理解越多，需要强记的就越少。
  • 勤做练习，写下心得。
  • 认真阅读“没有蠢问题”单元
  • 将阅读本书作为睡前最后一件事，或者至少当作睡前最后一件具有挑战性的事。
  • 谈论它，大声谈论它。
  • 喝水，多喝水。
  • 倾听大脑的声音。
  • 用心感受！
  • 动手设计网站吧！

一.建立美观网页

画出蓝图是第一要务

信息架构——IA

注释：Information Architecture,简称IA，是一种过程，主要将网站分解为多个块(chunk)，然后依内容块的逻辑关系分 等级 地将他们组织起来。

典型网站蓝图信息架构:

产品迭代(iterative)

注释：一种设计方法论，在工作过程中测试、分析，然后进一步琢磨作品的原型并加以改进，过程中的每一个阶段都会重复每一个步骤，如此循环，直至得到客户满意的结果。

step(简易)：

  1. 准备阶段：信息架构、分镜表、蓝图
  2. 导航系统：基于IA的逻辑从属关系
  3. 网页布局：前期纸笔作业，后期HTML、CSS
  4. 网站写作：填写具有“可扫读性”内容

二.前期制作

视觉隐喻

注释：与内容相关事物视觉元素，例如图片、字形、色彩以强化主题。

tips：

  1. 字形
  2. 调色板
  3. 七牛图床

三.组织网站

HF：The everything is for customer

具体流程

  1. 收集所有内容
  2. 头脑风暴，想出与主题相关视觉隐喻
  3. 开发视觉隐喻并思考布局
  4. 建立分镜表(纸上作业)
  5. 卡片分类、创建IA图
  6. HTML、CSS..

卡片分类：简短且具描述性

注意：

  1. 对于链接元素，必须从<a>标签设定色彩。
  2. 区别 display:inline 与 float:left 。

四.布局与设计

设计为客户所用！

  1. 建设角色：用户代表；
  2. 屏幕分辨率：择理想、兼多数；
  3. 黄金比例(Golden Ratio)：R=0.62 近似 3:1 。

设技要点：

  1. margin:0 auto; // 默认自适屏设定。
  2. 分辨率越低，事物显得越大，占用面积更多。
  3. 黄金比例更赏心悦目，后继出 三分法 同理

五.运用色彩设计

色彩具有强烈的情绪效应

step: 三色方案 ——> 色轮 ——> 调色板（kuler）

tips：

icon ：图标
hex ：16进制
saturation ：饱和度
contrasting ：对比
dominant color ：主色

六.智慧导航

良好的导航第一步就是良好的IA

  1. 语义：逻辑关系清晰，从属条理正确。
  2. 视觉：视觉感官良好，色彩搭配与主题呼应。

名称的魔力：

  1. 保持名称简短。
  2. 保持名称具有可描述性。

tips： 可用卡片分类法。

更便利布局设定：

为文字配上图标，而不是图标配上文字。

七.网站写作

问题在于文字

编号用于功能上的重要性而非尺寸

  1. <h1>、<h2>用于主标题；
  2. 子标题采用较低层次；
  3. css可以把较低层次标题设为较大字号，但注意语义正确，规范使用。

字体

Serif(衬线字体)：主笔画延伸

  • 用于：大标题
  • 字体：Times New Roman

Sans-Serif(非衬线字体)：字体均衡

  • 用于：主要内容
  • 字体：Helvetica , Arial

tips：标题比文章更具可扫读性

八.可访问性

WCAG

注释：全称 Web Content Accessibility Guideline (Web 内容可访问方针)。

Audio-2-Go案例

  • 盲人：<img>中用Alt标签，longdecs进行长文介绍。
  • 无鼠标：添加tabindex=”num”,编辑序列。

九.倾听用户的心声

问卷调查

  1. 较少行话 = 较佳理解程度 = 准确回答
  2. 别问假设题目
  3. 只问一个问题

十.保持网站新鲜

  1. 保证内容新鲜以保持用户回访率
  2. Web设计在于演变而非革命
    a. css演变、Web设计
    b. javascript (栗子：jQuery) 为网站添加特效
  3. 使用worldpress加入博客功能

十一.Web设计这一行

Web设计师类型

  1. 前台设计师：HTML、CSS、程序美术结合
  2. 信息架构师：系统导航
  3. 用户体验师：性能测试
  4. 图像设计师：开发图形元素
  5. 程序设计师：后台、编写Javascript、Ajax
  6. 文案编辑：文案

案例

  1. 新的潜在客户：Foo Bar
  2. Foo Bar对界面的要求、报价的期望
  3. 为Foo Bar建立快速模型

建站实际需求

  1. 初试时间投资 —— 开会、沟通、了解客户需求
  2. 概念构想时间 —— 设计信息架构、制作分镜表
  3. 草图与设计修订 —— 与客户沟通协调、进行初期产品迭代
  4. 制作时间 —— 编写 HTML、CSS、Javascript
  5. 其他费用 —— 交通、软件、字体、图像
  6. 计算总价 —— 所有成本累计、并要求预付25%定金
  7. 版权保护 —— 作品授权 Creative Commons

    d. 国旗代表语言不礼貌

  2. 针对移动设备设计

  3. 链接与指向网页的名称相符

  4. 对比是布局的策略基础
