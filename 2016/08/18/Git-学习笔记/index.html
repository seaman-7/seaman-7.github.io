5. 多人协作

  1. 操作步骤，先github上新建dev分支，再本地 git pull,后git checkout -b dev origin/dev，完成后git push origin dev

  2. 使用 git remote -v #查看远程库信息

  3. 本地新建的分支如果不推送到远程，对其他人就是不可见的；

温馨提示 ：

  • 使用git push origin branch-name #从本地推送分支，如果推送失败，先用git pull抓取远程的新提交；

  • 使用git checkout -b branch-name origin/branch-name #在本地创建和远程分支对应的分支，本地和远程分支的名称最好一致；

  • 使用git branch –set-upstream branch-name origin/branch-name #建立本地分支和远程分支的关联，

  • 使用git pull #从远程抓取分支，如果有冲突，要先处理冲突。
